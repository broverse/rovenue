# SDK M7.1 — iOS External Publish Prep (CocoaPods Trunk Infra)

**Date:** 2026-05-28
**Author:** vfurkanguner
**Status:** Draft

## Goal

Make `Rovenue` (the M3 Swift façade pod) **ready to push** to CocoaPods Trunk. Land the podspec shape, `librovenue.a` packaging pipeline, release script, CI lint gate, and flip the M6 config plugin's iOS default to Trunk-pod-first. **No actual `pod trunk push` happens in this milestone.** Once these pieces land, a human runs `pod trunk register` + `pod trunk push` manually.

After M7.1 lands and the first manual push happens, any external (non-monorepo) Expo / bare RN consumer that adds `@rovenue/react-native-sdk` to their `app.json` plugins gets a working iOS build with zero additional config — the plugin's default branch resolves `Rovenue` from Trunk.

Monorepo consumers (our `examples/sample-rn-expo/`) keep working because they pass an explicit `rovenueSwiftPath` option that selects the M6 `:path =>` branch.

## Non-Goals

- **Actual `pod trunk push`.** Interactive auth + irreversible — operator runs it manually after this milestone lands. The release script prints the command but does not execute it.
- **Maven Central publish / Android external-consumer plugin path.** Separate milestone (M7.2). Android plugin keeps the M6 hard-coded monorepo path; external Android consumers stay broken.
- **Rust tracing → JS forwarding.** Separate milestone (M8).
- **Automatic version bumping or changelog generation.** Operator hand-edits the podspec version.
- **Multi-architecture `librovenue.a`.** M7.1 ships a single-architecture (arm64) static library — sufficient for device builds and Apple Silicon simulators. x86_64 simulator support is an Open Question for follow-up.
- **CI-driven release.** GitHub Releases creation uses the `gh` CLI from a developer machine, not a workflow.
- **Directory rename** (`packages/sdk-rn/` → `packages/react-native-sdk/`). Out of scope as in M6.

## Decisions

| Concern | Choice | Reason |
|---|---|---|
| Podspec source | `:http` pointing at a GitHub Release `.zip`, pinned by `sha256` | Standalone tarball; no monorepo dependency at install time; lint validates without git checkout; future cache-friendly |
| `librovenue.a` packaging | Built locally by `release-pod.sh`, zipped together with Swift sources, uploaded as GH Release asset | Avoids committing a ~10 MB binary; release artifacts are immutable post-upload; lint can fetch the same tarball CI will fetch |
| Architecture | Single-arch arm64 (device + Apple Silicon sim) | M3 baseline; x86_64 simulators are an edge case for older Macs. Open Question for fat-binary follow-up |
| Trunk push trigger | Manual operator step after milestone merge | Trunk auth is interactive; one-way step; not worth automating in this milestone |
| Plugin iOS default | Trunk pod (`pod 'Rovenue', '~> 0.1'`); monorepo opts in via `rovenueSwiftPath` | Default serves the npm-public consumer; monorepo is the special case; no future "flip the default" debt |
| Plugin Android default | Unchanged from M6 (hard-coded monorepo `includeBuild`) | Out of scope; documented in plugin README |
| Sample app | Explicit `rovenueSwiftPath` option in `app.json` plugins entry | Exercises the override branch; serves as living documentation |
| Plugin testing | Vitest unit tests for `withRovenueIos` (Trunk default + path override + idempotency) | Plugin logic was untested in M6; now testable without `expo prebuild` |
| CI lint gate | `.github/workflows/sdk-swift-pod-lint.yml` — runs `pod spec lint --allow-warnings` on PRs that touch `packages/sdk-swift/**` or the podspec | Catches podspec regressions early; allow-warnings because vendored static libs flag warnings cocoapods can't avoid |
| Parity script | Best-effort `pod spec lint` step (skips if `pod` not on PATH) | Same pattern as M6 Kotlin / native-build skip |

## Architecture

Four working units. Each has a single owner and a tested boundary.

### 1. `packages/sdk-swift/Rovenue.podspec`

Trunk-publishable Pod spec. Source is an HTTP URL to a GitHub Release `.zip` containing the Swift sources + `librovenue.a`, pinned by sha256.

```ruby
Pod::Spec.new do |s|
  s.name             = 'Rovenue'
  s.version          = '0.1.1'
  s.summary          = 'Rovenue Swift façade'
  s.homepage         = 'https://rovenue.dev'
  s.license          = { :type => 'AGPL-3.0' }
  s.authors          = 'Rovenue'
  s.platforms        = { :ios => '13.0' }
  s.swift_version    = '5.9'
  s.source           = {
    :http   => "https://github.com/rovenue/rovenue/releases/download/sdk-swift-v#{s.version}/Rovenue-#{s.version}.zip",
    :sha256 => '<SHA256_PLACEHOLDER_REPLACED_BY_RELEASE_SCRIPT>'
  }
  s.source_files       = 'Sources/Rovenue/**/*.swift'
  s.vendored_libraries = 'Sources/Rovenue/librovenue.a'
end
```

Version bumps from M6's `0.1.0` to `0.1.1` to signal the source-format change (path → http). M6 monorepo consumers still get `0.1.0` semantics via the `:path =>` plugin branch (which does not consult the `s.source` field).

### 2. `packages/sdk-swift/scripts/release-pod.sh`

One-shot release pipeline. Bash, fail-fast (`set -euo pipefail`). Steps:

1. **Preflight.** Verify working tree clean, branch is `main`, `gh` + `pod` + `shasum` on PATH, `gh auth status` OK.
2. **Read version** from `Rovenue.podspec` (`grep s.version`).
3. **Build.** Run `packages/core-rs/scripts/build-bindings.sh` (produces `target/release/librovenue.a` → already copied to `packages/sdk-swift/Sources/Rovenue/librovenue.a` by that script's existing logic).
4. **Stage.** `mkdir build/Rovenue-$VERSION && cp -R Sources Rovenue.podspec build/Rovenue-$VERSION/`.
5. **Zip.** `(cd build && zip -r Rovenue-$VERSION.zip Rovenue-$VERSION)`.
6. **SHA256.** `shasum -a 256 build/Rovenue-$VERSION.zip | awk '{print $1}'`.
7. **Upload.** `gh release create sdk-swift-v$VERSION build/Rovenue-$VERSION.zip --title "sdk-swift v$VERSION" --notes "..."` — fails if the release already exists (operator must bump version).
8. **Patch podspec** in-place: replace the `:sha256 => '...'` line with the computed sha. Commit the change as `chore(sdk-swift): pin Rovenue podspec sha256 for v$VERSION`.
9. **Lint.** `pod spec lint Rovenue.podspec --allow-warnings`. If lint fails, revert the sha commit and exit non-zero.
10. **Print push command.** Echo `pod trunk push packages/sdk-swift/Rovenue.podspec --allow-warnings` for the operator to run manually. Never execute it.

Flags:
- `--dry-run` — skip steps 7, 8, 9, 10 (only build + zip + sha calculation; useful for local validation)
- `--skip-upload` — skip step 7 only (use when release already exists; recomputes sha + relints)

### 3. `.github/workflows/sdk-swift-pod-lint.yml`

PR gate. Runs only when `packages/sdk-swift/**` or `.github/workflows/sdk-swift-pod-lint.yml` changes. macOS runner.

```yaml
name: sdk-swift pod lint
on:
  pull_request:
    paths:
      - 'packages/sdk-swift/**'
      - '.github/workflows/sdk-swift-pod-lint.yml'
jobs:
  lint:
    runs-on: macos-14
    steps:
      - uses: actions/checkout@v4
      - run: pod spec lint packages/sdk-swift/Rovenue.podspec --allow-warnings --skip-tests
```

`--skip-tests` because the lint pod tries to compile a synthetic test app, which requires fetching the `:http` source — that fetch can only succeed for tags that already exist on GitHub Releases. On PRs that bump the version (sha mismatch is expected pre-release), the lint will fail; the release-pod.sh flow handles relint after the upload. The CI gate covers podspec-only changes for already-released versions.

### 4. `packages/sdk-rn/plugin/withRovenueIos.ts`

Default flips to Trunk-pod; `rovenueSwiftPath` opt selects the M6 path-based branch.

```ts
type Options = { rovenueSwiftPath?: string } | undefined;

export const withRovenueIos: ConfigPlugin<Options> = (config, opts) => {
  return withDangerousMod(config, ["ios", async (cfg) => {
    const podfile = path.join(cfg.modRequest.platformProjectRoot, "Podfile");
    if (!fs.existsSync(podfile)) return cfg;

    const contents = fs.readFileSync(podfile, "utf8");
    if (contents.includes("pod 'Rovenue'")) return cfg; // idempotent

    const podLine = opts?.rovenueSwiftPath
      ? `  pod 'Rovenue', :path => '${opts.rovenueSwiftPath}'`
      : `  pod 'Rovenue', '~> 0.1'`;

    const patched = contents.replace(
      /(target\s+['"][^'"]+['"]\s+do)/,
      `$1\n${podLine}`,
    );
    fs.writeFileSync(podfile, patched);
    return cfg;
  }]);
};
```

The `pod 'Rovenue'` idempotency check covers both branches (it matches the prefix regardless of `:path =>` or version constraint).

### 5. `examples/sample-rn-expo/app.json` (single-line change)

```json
"plugins": [
  ["@rovenue/react-native-sdk", { "rovenueSwiftPath": "../../../packages/sdk-swift" }]
]
```

The path is relative to the prebuild output's `ios/Podfile`. From `examples/sample-rn-expo/ios/` three `..` segments land at the repo root, then `packages/sdk-swift`. Identical to the M6 hard-coded default — only the location of the literal moves (plugin source → consumer config).

## Testing Strategy

### Unit tests (new)

`packages/sdk-rn/plugin/__tests__/withRovenueIos.test.ts` — vitest, runs in the existing sdk-rn test suite (the controller-verified 50 passing).

Tests:
1. **Trunk default** — no opts → Podfile contains `pod 'Rovenue', '~> 0.1'`
2. **Path override** — `{ rovenueSwiftPath: '../foo' }` → Podfile contains `pod 'Rovenue', :path => '../foo'`
3. **Idempotency** — running the plugin twice on the same Podfile produces identical output (no duplicate pod line)
4. **No Podfile** — if `platformProjectRoot/Podfile` doesn't exist, returns config unchanged (no crash)
5. **No `target` block** — if Podfile has no `target ... do`, returns config with Podfile unchanged (regex no-match)

Fixture: `packages/sdk-rn/plugin/__tests__/_fixturePodfile.ts` exports a minimal Podfile string and a helper that simulates `withDangerousMod`'s context (writes Podfile to a temp dir, runs the mod, reads back).

Total new tests: 5. Suite goes 50 → 55.

### CI lint

`pod spec lint` runs only when relevant paths change. It's NOT a blocker for unrelated PRs.

### Parity script

`scripts/sdk-parity.sh` gains a best-effort `pod spec lint` step:

```bash
if command -v pod >/dev/null 2>&1; then
  pod spec lint packages/sdk-swift/Rovenue.podspec --allow-warnings --skip-tests >/tmp/rovenue-podspec-lint.log 2>&1
  echo "  ✓ Rovenue podspec lints"
else
  echo "  ~ cocoapods unavailable — podspec lint skipped"
fi
```

Same skip pattern as M6's Kotlin / iOS-sample sections.

### Release script verification

The release script ships with a `--dry-run` flag. Local developer can run `./packages/sdk-swift/scripts/release-pod.sh --dry-run` to validate build + zip + sha calculation without touching GitHub Releases or git.

## Release Workflow (operator runbook)

After M7.1 merges, the first Trunk push is a manual flow:

1. Operator runs `pod trunk register your-email@rovenue.dev "Your Name"` (one-time per machine).
2. From repo root: `./packages/sdk-swift/scripts/release-pod.sh`
3. Review the printed `pod trunk push ...` command, then run it.
4. Verify on `https://cocoapods.org/pods/Rovenue`.

Subsequent releases: bump the version in `Rovenue.podspec`, then re-run the script.

## Migration / Compatibility

The plugin default flip is a **soft breaking change** for any consumer relying on the implicit M6 monorepo path. Currently only `examples/sample-rn-expo/` is in that bucket — covered by the explicit-option update in this same milestone.

The podspec source format change (`:path` → `:http`) does NOT affect the plugin's `:path =>` branch, which constructs its own `:path =>` directive in the consumer's Podfile and never reads `s.source`. Monorepo consumers see no behavioral change.

External consumers (zero before M7.1, also zero immediately after — Trunk push hasn't happened yet) gain a working code path that activates the moment the first manual `pod trunk push` succeeds.

## Open Questions / Deferred

1. **Multi-architecture `librovenue.a`.** Single-arch arm64 covers device + Apple Silicon simulators. x86_64 simulators (Intel Mac developers) will fail to link. Fix: extend `release-pod.sh` to build both arches and `lipo` them into a fat binary, or distribute an XCFramework. Deferred until an Intel-Mac user reports the gap.
2. **Trunk auth in CI.** First push is manual. Future automation needs Trunk session token stored as a GitHub secret + a release workflow trigger. Deferred until first push proves the manual flow works.
3. **`gh release create` for tag bumps.** If the version in `Rovenue.podspec` already has a corresponding GH Release, the script exits non-zero. An override (`--force-recreate`) is possible but not added — operator deletes the release manually if they really need to replay.
4. **Maven Central + Android external plugin path.** Separate milestone (M7.2). Track separately.
5. **`Rovenue 0.1.1` synchronization with M3 Swift's internal version.** M3's `sdkVersion()` Rust binding returns the Cargo `librovenue` version (`0.0.1`). The podspec version (`0.1.1`) is independent — this drift is intentional (podspec versions track the publishable artifact's shape changes, not the Rust core). Document in plugin README to avoid confusion.
