The SDK drives the full purchase lifecycle — fetching products from the store,
presenting them, triggering StoreKit 2 / Play Billing, and validating the
receipt server-side — all in a single `purchase(pkg)` call. Your app only
decides *which* package to offer.

## Fetch offerings

Offerings are configured in the Rovenue dashboard. Fetch them at app launch or
when your paywall opens:

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import {
      Rovenue,
      PurchaseCancelledError,
      PurchasePendingError,
    } from '@rovenue/react-native-sdk';

    const offerings = await Rovenue.getOfferings();
    // offerings.current — the default offering (or null if none configured)
    // offerings.all    — Record<string, Offering> for all offerings
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    let offerings = try await Rovenue.shared.getOfferings()
    // offerings.current — the default Offering? 
    // offerings.all     — [String: Offering]
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    val offerings = Rovenue.shared.getOfferings()
    // offerings.current — the default Offering?
    // offerings.all     — Map<String, Offering>
    ```
  </Tab>
</Tabs>

Each `Offering` contains a list of `Package` objects. Each `Package` wraps a
`StoreProduct` with localised pricing from the store:

```ts
type StoreProduct = {
  id: string;
  type: 'subscription' | 'consumable' | 'non_consumable';
  displayName: string;
  priceString: string; // e.g. "$4.99"
  price: number;
  currencyCode: string;
};
```

## Purchase a package

Pass the chosen `Package` to `Rovenue.purchase()`. The SDK triggers the native
store sheet (StoreKit 2 on iOS, Play Billing on Android), waits for the
transaction to complete, and validates it server-side before resolving. Your
app receives updated entitlements and credit balance in the result — no
additional polling needed.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import {
      Rovenue,
      PurchaseCancelledError,
      PurchasePendingError,
      ProductNotAvailableError,
      StoreProblemError,
    } from '@rovenue/react-native-sdk';

    const offerings = await Rovenue.getOfferings();
    const pkg = offerings.current?.packages[0];

    if (pkg) {
      try {
        const result = await Rovenue.purchase(pkg);
        // result.entitlements and result.creditBalance are already up to date
        console.log('Purchase succeeded:', result.productId);
      } catch (e) {
        if (e instanceof PurchaseCancelledError) {
          // User dismissed the store sheet — no action needed
        } else if (e instanceof PurchasePendingError) {
          // Ask-to-Buy / deferred — inform the user and wait for a change event
        } else {
          throw e;
        }
      }
    }
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    let offerings = try await Rovenue.shared.getOfferings()
    if let pkg = offerings.current?.packages.first {
      do {
        let result = try await Rovenue.shared.purchase(pkg)
        print("Purchase succeeded: \(result.productId)")
        // result.entitlements and result.creditBalance are already up to date
      } catch is PurchaseCancelledError {
        // User dismissed the store sheet — no action needed
      } catch is PurchasePendingError {
        // Ask-to-Buy / deferred — inform the user and wait for a change event
      }
    }
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    val offerings = Rovenue.shared.getOfferings()
    val pkg = offerings.current?.packages?.firstOrNull()

    if (pkg != null) {
      try {
        val result = Rovenue.shared.purchase(pkg)
        println("Purchase succeeded: ${result.productId}")
        // result.entitlements and result.creditBalance are already up to date
      } catch (e: PurchaseCancelledError) {
        // User dismissed the store sheet — no action needed
      } catch (e: PurchasePendingError) {
        // Ask-to-Buy / deferred — inform the user and wait for a change event
      }
    }
    ```
  </Tab>
</Tabs>

`purchase()` resolves with a `PurchaseResult` after server-side validation
completes:

```ts
type PurchaseResult = {
  entitlements: Entitlement[];
  creditBalance: number;
  productId: string;
  storeTransactionId: string;
};
```

Rovenue decides server-side — based on your product configuration in the
dashboard — whether a product grants an entitlement (subscriptions, one-time
unlocks) or adds credits (consumables). Both product types go through the same
`purchase(pkg)` call.

<Callout type="warn">
**Possible errors**

- `PurchaseCancelledError` — the user dismissed the store sheet. Safe to ignore.
- `PurchasePendingError` — Ask-to-Buy (iOS) or a pending Play Billing transaction.
  Show a "waiting for approval" message and listen for a change event.
- `ProductNotAvailableError` — the product ID is not available in this store environment.
- `StoreProblemError` — an unexpected StoreKit / Play Billing error. Retry or surface to the user.
</Callout>

## Restore purchases

Call `restorePurchases()` to re-validate all prior transactions for the current
user (e.g. after a reinstall or platform restore prompt):

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    await Rovenue.restorePurchases();
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    try await Rovenue.shared.restorePurchases()
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    Rovenue.shared.restorePurchases()
    ```
  </Tab>
</Tabs>

On completion the SDK emits a change event with the refreshed entitlement
state. See [Reacting to changes](/docs/guides/reacting-to-changes).
