Every Rovenue SDK error originates from the same Rust-core `RovenueError` type and is mapped to an idiomatic type on each platform:

- **React Native** тАФ thrown as `RovenueError` (a class extending `Error`) with a `kind` discriminant. Use `e.kind === "..."` to branch.
- **Swift** тАФ thrown as `RovenueError` (a struct conforming to `LocalizedError`) with a `kind: ErrorKind` discriminant. Catch with `catch let e as RovenueError`.
- **Kotlin** тАФ thrown as `RovenueException` (sealed class) with a `kind: ErrorKind` discriminant. Catch by `kind` or the sealed base.

## Migrating from 0.15.x тЖТ 0.16.0

<Callout type="warn">
**Breaking change in 0.16.0** тАФ the per-type error subclasses and Swift `Rovenue.Error` enum cases are replaced by a single `RovenueError`/`RovenueException` with a `kind` string discriminant, and `.pending` purchases are now a non-throwing `Deferred` outcome.
</Callout>

### React Native: `instanceof` тЖТ `e.kind`

```ts
// Before (0.15.x)
import { InsufficientCreditsError, NetworkUnavailableError } from '@rovenue/react-native-sdk';
try {
  await Rovenue.purchase(pkg);
} catch (e) {
  if (e instanceof InsufficientCreditsError) { /* тАж */ }
  if (e instanceof NetworkUnavailableError) { /* тАж */ }
}

// After (0.16.0)
import { RovenueError } from '@rovenue/react-native-sdk';
try {
  const result = await Rovenue.purchase(pkg);
  if (result.isDeferred) {
    // Ask-to-Buy / parental controls тАФ not an error, poll or wait for the
    // PURCHASES_UPDATED listener before granting entitlements.
    showToast('Your purchase is awaiting approval.');
    return;
  }
} catch (e) {
  if (e instanceof RovenueError) {
    if (e.kind === 'InsufficientCredits') { /* тАж */ }
    if (e.kind === 'NetworkUnavailable') { /* тАж */ }
  }
}
```

### Swift: `Rovenue.Error` enum тЖТ `RovenueError` struct

```swift
// Before (0.15.x)
do {
  try await Rovenue.shared.purchase(pkg)
} catch Rovenue.Error.purchasePending { showToast("Awaiting approval.") }
  catch Rovenue.Error.insufficientCredits { /* тАж */ }

// After (0.16.0)
do {
  let result = try await Rovenue.shared.purchase(pkg)
  if result.isDeferred { showToast("Awaiting approval."); return }
} catch let e as RovenueError {
  switch e.kind {
  case .insufficientCredits: /* тАж */
  default: throw e
  }
}
```

### Kotlin: standalone exceptions тЖТ `RovenueException.kind`

```kotlin
// Before (0.15.x)
try {
  Rovenue.shared.purchase(activity, pkg)
} catch (e: PurchasePendingException) { showToast("Awaiting approval.") }
  catch (e: PurchaseCancelledException) { /* user cancelled */ }

// After (0.16.0)
val outcome = Rovenue.shared.purchase(activity, pkg)
if (outcome.isDeferred) { showToast("Awaiting approval."); return }
// Errors:
try {
  Rovenue.shared.purchase(activity, pkg)
} catch (e: RovenueException) {
  when (e.kind) {
    ErrorKind.PURCHASE_CANCELED -> { /* user cancelled тАФ no action */ }
    else -> throw e
  }
}
```

### Handling the Deferred purchase state

`purchase()` no longer throws for Ask-to-Buy / parental-control holds. Instead, it returns a `StorePurchaseOutcome` with `isDeferred = true`. You should:

1. Show a friendly message (e.g. "Your purchase is awaiting approval from your family organizer.").
2. Do **not** grant entitlements тАФ entitlements are only granted after the purchase completes asynchronously.
3. Listen for the `PURCHASES_UPDATED` notification (RN: `Rovenue.addPurchasesUpdatedListener`; Swift: `RovenueDelegate.purchasesDidUpdate`; Kotlin: `PurchasesUpdatedListener`) to know when the deferred purchase resolves.

## Error table

| When it fires | React Native (class) | Swift (enum case) | Kotlin (exception) |
|---------------|---------------------|-------------------|--------------------|
| SDK not configured тАФ `configure()` was not called before accessing `Rovenue.shared` | `NotConfiguredError` | `.notConfigured` | `RovenueException.NotConfigured` |
| API key is missing, malformed, or rejected by the server | `InvalidApiKeyError` | `.invalidApiKey` | `RovenueException.InvalidApiKey` |
| An argument was invalid тАФ e.g. an empty string passed to `identify()` | `InvalidArgumentError` | `.invalidArgument` | `RovenueException.InvalidArgument` |
| Server returned a non-recoverable error response | `ServerError` | `.serverError` | `RovenueException.ServerException` |
| Device has no network connectivity | `NetworkUnavailableError` | `.networkUnavailable` | `RovenueException.NetworkUnavailable` |
| Request exceeded the allowed time limit | `TimeoutError` | `.timeout` | `RovenueException.Timeout` |
| Too many requests тАФ SDK is being rate-limited; back off and retry | `RateLimitedError` | `.rateLimited` | `RovenueException.RateLimited` |
| Local persistent storage (MMKV) read/write failure | `StorageError` | `.storage` | `RovenueException.Storage` |
| Subscriber record not found on the server | `UserNotFoundError` | `.userNotFound` | `RovenueException.UserNotFound` |
| Returned by the server when a virtual-currency spend transaction has insufficient balance (server-side API only тАФ no client SDK method throws this) | `InsufficientCreditsError` | `.insufficientCredits` | `RovenueException.InsufficientCredits` |
| Entitlement exists but is not currently active | `EntitlementInactiveError` | `.entitlementInactive` | `RovenueException.EntitlementInactive` |
| Receipt was already recorded for this transaction (idempotency deduplication) | `DuplicatePurchaseError` | `.duplicatePurchase` | `RovenueException.DuplicatePurchase` |
| Receipt could not be validated by the store or server | `ReceiptInvalidError` | `.receiptInvalid` | `RovenueException.ReceiptInvalid` |
| Unexpected internal SDK error | `InternalError` | `.internalError` | `RovenueException.Internal` |
| User cancelled the store purchase sheet | `PurchaseCancelledError` | `.purchaseCancelled` | `PurchaseCancelledException` |
| Purchase is pending approval (Ask-to-Buy / parental controls) | `PurchasePendingError` | `.purchasePending` | `PurchasePendingException` |
| The selected product is not available in the user's storefront | `ProductNotAvailableError` | `.productNotAvailable` | `ProductNotAvailableException` |
| The store returned a non-specific billing or connectivity error | `StoreProblemError` | `.storeProblem` | `StoreProblemException` |

<Callout type="warn">
**`InvalidApiKeyError` vs `InvalidArgumentError`**

`InvalidApiKeyError` fires when the SDK detects that your API key is malformed (wrong prefix) at `configure()` time, or when the server rejects it on the first network request. It indicates a **configuration problem** тАФ check your dashboard key.

`InvalidArgumentError` fires when your code passes an invalid value to a method at call time тАФ for example, an empty string to `identify()`. It indicates a **programming error** in your call site.
</Callout>

## Handling errors

All throwing SDK methods should be wrapped in try/catch. The recommended pattern is to handle the errors you expect explicitly, then re-throw anything unexpected so it surfaces in your crash reporting.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import {
      Rovenue,
      PurchaseCancelledError,
      PurchasePendingError,
      ProductNotAvailableError,
      ReceiptInvalidError,
      NetworkUnavailableError,
    } from '@rovenue/react-native-sdk';

    // --- Purchase flow ---
    try {
      const result = await Rovenue.purchase(pkg);
      // success
    } catch (e) {
      if (e instanceof PurchaseCancelledError) {
        // User dismissed the store sheet тАФ no action needed
      } else if (e instanceof PurchasePendingError) {
        // Ask-to-Buy тАФ inform the user to wait for parental approval
        showToast('Your purchase is awaiting approval.');
      } else if (e instanceof ProductNotAvailableError) {
        // Product not available in this storefront
        showToast('This product is not available in your region.');
      } else if (e instanceof ReceiptInvalidError) {
        // Receipt validation failed тАФ offer a retry
        showRetryPrompt();
      } else if (e instanceof NetworkUnavailableError) {
        showToast('No internet connection. Please try again.');
      } else {
        throw e; // re-throw unexpected errors
      }
    }
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    import Rovenue

    // --- Purchase flow ---
    do {
      let result = try await Rovenue.shared.purchase(pkg)
      // success
    } catch Rovenue.Error.purchaseCancelled {
      // User dismissed the store sheet тАФ no action needed
    } catch Rovenue.Error.purchasePending {
      // Ask-to-Buy тАФ inform the user to wait for parental approval
      showToast("Your purchase is awaiting approval.")
    } catch Rovenue.Error.productNotAvailable {
      showToast("This product is not available in your region.")
    } catch Rovenue.Error.receiptInvalid {
      // Receipt validation failed тАФ offer a retry
      showRetryPrompt()
    } catch Rovenue.Error.networkUnavailable {
      showToast("No internet connection. Please try again.")
    } catch {
      throw error // re-throw unexpected errors
    }
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    import dev.rovenue.sdk.Rovenue
    import dev.rovenue.sdk.generated.RovenueException
    import dev.rovenue.sdk.PurchaseCancelledException
    import dev.rovenue.sdk.PurchasePendingException
    import dev.rovenue.sdk.ProductNotAvailableException

    // --- Purchase flow ---
    try {
        // `activity` is your current Activity тАФ Play Billing requires it.
        val result = Rovenue.shared.purchase(activity, pkg)
        // success
    } catch (e: PurchaseCancelledException) {
        // User dismissed the store sheet тАФ no action needed
    } catch (e: PurchasePendingException) {
        // Ask-to-Buy тАФ inform the user to wait for parental approval
        showToast("Your purchase is awaiting approval.")
    } catch (e: ProductNotAvailableException) {
        showToast("This product is not available in your region.")
    } catch (e: RovenueException.ReceiptInvalid) {
        showRetryPrompt()
    } catch (e: RovenueException.NetworkUnavailable) {
        showToast("No internet connection. Please try again.")
    } catch (e: RovenueException) {
        throw e // re-throw unexpected SDK errors
    }
    ```
  </Tab>
</Tabs>

<Callout type="info">
`PurchaseCancelledError` is **not an error** from the user's perspective. It fires whenever the store sheet is dismissed without a completed payment тАФ including when the user deliberately taps Cancel. Do not show an error dialog for this case.
</Callout>
