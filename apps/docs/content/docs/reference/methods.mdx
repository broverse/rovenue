## Configuration

### `configure`

Initialise the SDK. Call this **once at app startup**, before any other Rovenue call. On React Native, pass a single config object. On Swift and Kotlin, use labelled parameters.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `apiKey` | `string` | Yes | Your project's public API key. Prefix `rov_pub_`. |
| `baseUrl` | `string` | No | API endpoint. Defaults to the hosted endpoint `https://api.rovenue.io`; override only when self-hosting (`https://`; `http://` allowed for `localhost` only). |
| `debug` | `boolean` | No | Enable verbose SDK logging. Defaults to `false`. |
| `appVersion` | `string` | No | App version string forwarded to the server for analytics. Defaults to the value reported by the OS. |

**Returns** — `void`

**Throws** — [`InvalidApiKeyError`](/docs/reference/errors#error-table) if the key format is invalid at call time (a malformed prefix). The key is not validated against the server until the first network request.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import { Rovenue } from '@rovenue/react-native-sdk';

    Rovenue.configure({
      apiKey: 'rov_pub_…',
      debug: __DEV__,
      appVersion: '3.1.0',
      // baseUrl: 'https://api.acme.com', // optional — only when self-hosting
    });
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    import Rovenue

    Rovenue.configure(
        apiKey: "rov_pub_…",
        debug: true,
        appVersion: "3.1.0"
        // baseUrl: "https://api.acme.com"  // optional — only when self-hosting
    )
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    import dev.rovenue.sdk.Rovenue

    Rovenue.configure(
        apiKey = "rov_pub_…",
        debug = BuildConfig.DEBUG,
        appVersion = BuildConfig.VERSION_NAME
        // baseUrl = "https://api.acme.com"  // optional — only when self-hosting
    )
    ```
  </Tab>
</Tabs>

<Callout type="info">
`configure` must be called before `Rovenue.shared` is accessed. Accessing any other method before `configure` throws [`NotConfiguredError`](/docs/reference/errors#error-table).
</Callout>

**See also:** [Configuring the SDK guide](/docs/guides/configuring)

### `identify`

Bind a backend user ID to the current install. Rovenue attaches the `appUserId` label to the `rovenueId` and, if that `appUserId` already exists on another install, transfers that install's purchases and credits to this device. The call is **optimistic** — local state updates immediately; the server sync completes in the background.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `appUserId` | `string` | Yes | Your backend's stable user identifier. Must be non-empty. |

**Returns** — `void`

**Throws** — [`InvalidArgumentError`](/docs/reference/errors#error-table) if `appUserId` is empty or blank.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    // After your auth flow resolves:
    await Rovenue.identify('user_123');
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    try await Rovenue.shared.identify("user_123")
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    Rovenue.shared.identify("user_123") // suspend fun
    ```
  </Tab>
</Tabs>

<Callout type="warn">
Use **opaque, unguessable** IDs — never an email address or username. Because `identify()` can transfer purchases between installs, a guessable ID is an account-takeover surface. See the [Identifying users guide](/docs/guides/identifying-users) for the recommended token-based flow.
</Callout>

**Errors:** [`InvalidArgumentError`](/docs/reference/errors#error-table)

## Virtual Currencies

### Method summary

| Method | Signature (RN) | Returns | Throws |
|--------|---------------|---------|--------|
| `virtualCurrencies` | `virtualCurrencies()` | `Promise<Record<string, number>>` | — |
| `virtualCurrency` | `virtualCurrency(code)` | `Promise<number>` | — |
| `refreshVirtualCurrencies` | `refreshVirtualCurrencies()` | `void` | [`NetworkUnavailableError`](/docs/reference/errors#error-table), [`ServerError`](/docs/reference/errors#error-table) |

React Native also exposes `useVirtualCurrencies()` and `useVirtualCurrency(code)` hooks. Swift uses `virtualCurrencyBalances() -> [String: Int64]` and `virtualCurrency(_ code:) -> Int64`. Kotlin uses `virtualCurrencyBalances(): Map<String, Long>` and `virtualCurrency(code): Long`.

`virtualCurrencies()` and `virtualCurrency(code)` resolve immediately from the local cache (no network round-trip), but are async — `await` them. `refreshVirtualCurrencies()` forces a server round-trip. **Spending virtual currency is server-side only** — call `POST /v1/virtual-currencies/:appUserId/:code/transactions` from your backend using your secret API key.

## Events

### `track`

Emit a generic event to the backend. The server forwards it to any configured integrations (e.g. Meta Conversions API, TikTok Events API). The call is **fire-and-forget** — the SDK does not wait for confirmation before returning.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `eventType` | `string` | Yes | The event name (e.g. `"purchase"`, `"lead"`, `"registration"`). |
| `params` | [`TrackParams`](#trackparams) | No | Optional event metadata. |

**`TrackParams`**

| Field | Type | Description |
|-------|------|-------------|
| `occurredAt` | `string` | ISO-8601 timestamp of when the event occurred. Defaults to now. |
| `subscriberId` | `string` | Override the subscriber scope. Defaults to the current user / anonymous install. |
| `productId` | `string` | Associated product identifier. |
| `amount` | `string` | Decimal string (e.g. `"9.99"`). |
| `currency` | `string` | ISO-4217 currency code (e.g. `"USD"`). |
| `eventSourceUrl` | `string` | URL of the page or screen where the event occurred. |
| `identityContext` | `object` | Hashed identity signals for ad-platform matching. Fields: `email`, `externalId`, `phone`, `ip`, `userAgent`, `firstName`, `lastName`, `city`, `countryCode`. |

**Returns** — `void`

**Throws** — nothing (errors are swallowed and logged internally)

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import { Rovenue } from '@rovenue/react-native-sdk';

    // Minimal — event type only:
    await Rovenue.track('lead');

    // With params:
    await Rovenue.track('purchase', {
      amount: '9.99',
      currency: 'USD',
      productId: 'com.example.pro_monthly',
    });

    // With identity context for ad-platform matching:
    await Rovenue.track('registration', {
      identityContext: {
        email: 'user@example.com',
        phone: '+15551234567',
      },
    });
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    // Minimal:
    await Rovenue.shared.track(eventType: "lead")

    // With params:
    await Rovenue.shared.track(
        eventType: "purchase",
        params: TrackParams(
            amount: "9.99",
            currency: "USD",
            productId: "com.example.pro_monthly"
        )
    )
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    // Minimal:
    Rovenue.shared.track(eventType = "lead")

    // With params:
    Rovenue.shared.track(
        eventType = "purchase",
        params = TrackParams(
            amount = "9.99",
            currency = "USD",
            productId = "com.example.pro_monthly"
        )
    )
    ```
  </Tab>
</Tabs>

<Callout type="info">
`track()` is a client-side convenience wrapper around `POST /v1/events`. For server-side event ingestion (e.g. from your backend after a payment processor webhook), call the HTTP endpoint directly with your secret API key.
</Callout>

### `purchase`

Trigger a native store sheet for the given package. The SDK handles the full purchase lifecycle: StoreKit 2 on iOS, Play Billing on Android, server-side receipt validation. Resolves with updated entitlements and credit balance — no additional polling needed.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `pkg` | [`Package`](/docs/reference/types#package) | Yes | A package from [`getOfferings()`](#getofferings). |

> **Android:** Kotlin requires the current `Activity` as the first argument — `purchase(activity, pkg)`. iOS (Swift) and React Native take only the package.

**Returns**

```ts
type PurchaseResult = {
  entitlements: Entitlement[];              // full set of active entitlements after the purchase
  virtualCurrencies: Record<string, number>; // virtual-currency balances after the purchase
  productId: string;                        // the store product ID purchased
  storeTransactionId: string;               // StoreKit 2 transaction ID or Play Billing order ID
};
```

→ Full type reference: [`PurchaseResult`](/docs/reference/types#purchaseresult)

**Throws** — see table below

| Error | When it fires |
|-------|--------------|
| [`PurchaseCancelledError`](/docs/reference/errors#error-table) | User dismissed the store sheet without completing payment |
| [`PurchasePendingError`](/docs/reference/errors#error-table) | Ask-to-Buy / parental approval pending — transaction not yet complete |
| [`ProductNotAvailableError`](/docs/reference/errors#error-table) | The product is not available in the user's storefront |
| [`StoreProblemError`](/docs/reference/errors#error-table) | The store returned a non-specific error (e.g. billing issue) |
| [`ReceiptInvalidError`](/docs/reference/errors#error-table) | Server-side receipt validation failed |

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import {
      Rovenue,
      PurchaseCancelledError,
      PurchasePendingError,
      ProductNotAvailableError,
      StoreProblemError,
      ReceiptInvalidError,
    } from '@rovenue/react-native-sdk';

    const offerings = await Rovenue.getOfferings();
    const pkg = offerings.current?.packages[0];

    if (pkg) {
      try {
        const result = await Rovenue.purchase(pkg);
        console.log('Purchase succeeded:', result.productId);
        // result.entitlements and result.virtualCurrencies are already up to date
      } catch (e) {
        if (e instanceof PurchaseCancelledError) {
          // User dismissed — no action needed
        } else if (e instanceof PurchasePendingError) {
          // Ask-to-Buy — tell the user to wait for approval
        } else if (e instanceof ReceiptInvalidError) {
          // Show a retry prompt
        } else {
          throw e;
        }
      }
    }
    ```
  </Tab>
  <Tab value="Swift">
    ```swift

      do {
        let result = try await Rovenue.shared.purchase(pkg)

      } catch Rovenue.Error.purchaseCancelled {
        // User dismissed — no action needed
      } catch Rovenue.Error.purchasePending {
        // Ask-to-Buy — tell the user to wait for approval
      } catch Rovenue.Error.receiptInvalid {
        // Show a retry prompt
      }
    }
    ```
  </Tab>
  <Tab value="Kotlin">

    ```kotlin
        // User dismissed — no action needed

        // Show a retry prompt
      }
    ```

### `restorePurchases`

Re-validate the user's existing store receipts against the Rovenue server and update the entitlement cache. Required by App Store guidelines to offer a "Restore purchases" button.

**Params** — none on iOS / React Native. **Android:** Kotlin requires the current `Activity` — `restorePurchases(activity)`.

**Returns** — `void`

## Lifecycle

### Method summary

| Method | Signature (RN) | Returns | Throws |
|--------|---------------|---------|--------|
| `setForeground` | `setForeground(isForeground)` | `void` | — |
| `shutdown` | `shutdown()` | `void` | — |

Wire these into your platform's app lifecycle hooks so the SDK can pause background work when the app is in the background and trigger a refresh when it returns to the foreground.

---

### `setForeground`

Notify the SDK of foreground/background transitions. When `true`, the SDK schedules a background entitlement sync to catch any server-side changes that occurred while the app was inactive.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `isForeground` | `boolean` | Yes | `true` when the app enters the foreground; `false` when it goes to the background. |

**Returns** — `void`

**Throws** — nothing

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">

    ```ts
    import { AppState } from 'react-native';

    });
    ```

  <Tab value="Swift">

## Logging

### `setLogHandler`

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|

```ts
type LogEntry = {
```

## Change Stream

Subscribe to real-time state change notifications from the SDK. The SDK fires an event whenever the entitlement cache, identity, or virtual-currency balances are updated. React to these events to keep your UI in sync without polling.

| Method / Property | Platform | Description |
|-------------------|----------|-------------|
| `addChangeListener(handler)` | React Native | Register a callback; returns an unsubscribe function. |
| `changes` | Swift | `AsyncStream<ChangeEvent>` — iterate with `for await`. |
| `changes` | Kotlin | `SharedFlow<ChangeEvent>` — collect with `.onEach { }.launchIn(scope)`. |

The [`ChangeEvent`](/docs/reference/types#changeevent) values:

| Value | When it fires |
|-------|--------------|
| `EntitlementsChanged` | After any entitlement cache refresh (purchase, `refreshEntitlements()`, foreground sync). |
| `IdentityChanged` | After `identify()` succeeds or `logOut()` resets the user. |
| `VirtualCurrenciesChanged` | After `refreshVirtualCurrencies()` or a server-side grant/spend that updates the cache. |

→ Full type reference: [`ChangeEvent`](/docs/reference/types#changeevent)

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import { Rovenue } from '@rovenue/react-native-sdk';

    const unsub = Rovenue.addChangeListener((event) => {
      switch (event) {
        case 'EntitlementsChanged':
          // Re-read entitlements and update gated UI
          break;
        case 'IdentityChanged':
          // User identity has changed — re-read currentUser()
          break;
        case 'VirtualCurrenciesChanged':
          // Re-read virtualCurrencies() and update the balance display
          break;
      }
    });

    // When your component unmounts:
    unsub();
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    // Typically in a Task {} inside your SwiftUI view or view model:
    for await event in Rovenue.shared.changes {
        switch event {
        case .entitlementsChanged:
            // Re-read entitlements and update gated UI
        case .identityChanged:
            // User identity has changed — re-read currentUser()
        case .virtualCurrenciesChanged:
            // Re-read virtualCurrencyBalances() and update the balance display
        }
    }
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    // In a ViewModel or LifecycleCoroutineScope:
    Rovenue.shared.changes
        .onEach { event ->
            when (event) {
                ChangeEvent.ENTITLEMENTS_CHANGED       -> { /* re-read entitlements */ }
                ChangeEvent.IDENTITY_CHANGED           -> { /* user identity changed */ }
                ChangeEvent.VIRTUAL_CURRENCIES_CHANGED -> { /* re-read balances */ }
            }
        }
        .launchIn(viewModelScope)
    ```
  </Tab>
</Tabs>

**See also:** [Reacting to changes guide](/docs/guides/reacting-to-changes)
