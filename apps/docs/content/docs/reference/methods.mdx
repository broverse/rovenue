## Configuration

### `configure`

Initialise the SDK. Call this **once at app startup**, before any other Rovenue call. On React Native, pass a single config object. On Swift and Kotlin, use labelled parameters.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `apiKey` | `string` | Yes | Your project's public API key. Prefix `rov_pub_`. |
| `baseUrl` | `string` | Yes | Edge endpoint. Use `https://edge.rovenue.app` for production. |
| `debug` | `boolean` | No | Enable verbose SDK logging. Defaults to `false`. |
| `appVersion` | `string` | No | App version string forwarded to the server for analytics. Defaults to the value reported by the OS. |

**Returns** — `void`

**Throws** — [`InvalidApiKeyError`](/docs/reference/errors#error-table) if the key format is invalid at call time (a malformed prefix). The key is not validated against the server until the first network request.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import { Rovenue } from '@rovenue/react-native-sdk';

    Rovenue.configure({
      apiKey: 'rov_pub_…',
      baseUrl: 'https://edge.rovenue.app',
      debug: __DEV__,
      appVersion: '3.1.0',
    });
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    import Rovenue

    Rovenue.configure(
        apiKey: "rov_pub_…",
        baseUrl: "https://edge.rovenue.app",
        debug: true,
        appVersion: "3.1.0"
    )
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    import com.rovenue.Rovenue

    Rovenue.configure(
        apiKey = "rov_pub_…",
        baseUrl = "https://edge.rovenue.app",
        debug = BuildConfig.DEBUG,
        appVersion = BuildConfig.VERSION_NAME
    )
    ```
  </Tab>
</Tabs>

<Callout type="info">
`configure` must be called before `Rovenue.shared` is accessed. Accessing any other method before `configure` throws [`NotConfiguredError`](/docs/reference/errors#error-table).
</Callout>

**See also:** [Configuring the SDK guide](/docs/guides/configuring)

### `identify`

Bind a backend user ID to the current install. Rovenue attaches the `appUserId` label to the `rovenueId` and, if that `appUserId` already exists on another install, transfers that install's purchases and credits to this device. The call is **optimistic** — local state updates immediately; the server sync completes in the background.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `appUserId` | `string` | Yes | Your backend's stable user identifier. Must be non-empty. |

**Returns** — `void`

**Throws** — [`InvalidArgumentError`](/docs/reference/errors#error-table) if `appUserId` is empty or blank.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    // After your auth flow resolves:
    await Rovenue.identify('user_123');
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    try await Rovenue.shared.identify("user_123")
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    Rovenue.shared.identify("user_123") // suspend fun
    ```
  </Tab>
</Tabs>

<Callout type="warn">
Use **opaque, unguessable** IDs — never an email address or username. Because `identify()` can transfer purchases between installs, a guessable ID is an account-takeover surface. See the [Identifying users guide](/docs/guides/identifying-users) for the recommended token-based flow.
</Callout>

**Errors:** [`InvalidArgumentError`](/docs/reference/errors#error-table)

### `consumeCredits`

Atomically deduct credits from the user's balance. The deduction is applied server-side and the updated balance is returned. If the user has insufficient credits, the call throws without modifying the balance.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `amount` | `number` | Yes | Number of credits to deduct. Must be a positive integer. |
| `description` | `string` | No | Human-readable label for this deduction (e.g. `"AI generation"`). Stored in the credit ledger for audit purposes. |

**Returns** — `number` (the new balance after deduction)

**Throws** — [`InsufficientCreditsError`](/docs/reference/errors#error-table) if the user has fewer credits than `amount`.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import { Rovenue, InsufficientCreditsError } from '@rovenue/react-native-sdk';

    try {
      const newBalance = await Rovenue.consumeCredits(10, 'AI generation');
      console.log(`Balance after: ${newBalance}`);
    } catch (e) {
      if (e instanceof InsufficientCreditsError) {
        // Prompt the user to purchase more credits
      } else {
        throw e;
      }
    }
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    do {
      let newBalance = try await Rovenue.shared.consumeCredits(
          amount: 10,
          description: "AI generation"
      )
      print("Balance after: \(newBalance)")
    } catch Rovenue.Error.insufficientCredits {
      // Prompt the user to purchase more credits
    }
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    try {
      val newBalance = Rovenue.shared.consumeCredits(
          amount = 10,
          description = "AI generation"
      )
      Log.d("App", "Balance after: $newBalance")
    } catch (e: RovenueException.InsufficientCredits) {
      // Prompt the user to purchase more credits
    }
    ```
  </Tab>
</Tabs>

**Errors:** [`InsufficientCreditsError`](/docs/reference/errors#error-table)

**See also:** [Credits guide](/docs/guides/credits)

### `purchase`

Trigger a native store sheet for the given package. The SDK handles the full purchase lifecycle: StoreKit 2 on iOS, Play Billing on Android, server-side receipt validation. Resolves with updated entitlements and credit balance — no additional polling needed.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `pkg` | [`Package`](/docs/reference/types#package) | Yes | A package from [`getOfferings()`](#getofferings). |

**Returns**

```ts
type PurchaseResult = {
  entitlements: Entitlement[];   // full set of active entitlements after the purchase
  creditBalance: number;         // total credit balance after the purchase
  productId: string;             // the store product ID purchased
  storeTransactionId: string;    // StoreKit 2 transaction ID or Play Billing order ID
};
```

→ Full type reference: [`PurchaseResult`](/docs/reference/types#purchaseresult)

**Throws** — see table below

| Error | When it fires |
|-------|--------------|
| [`PurchaseCancelledError`](/docs/reference/errors#error-table) | User dismissed the store sheet without completing payment |
| [`PurchasePendingError`](/docs/reference/errors#error-table) | Ask-to-Buy / parental approval pending — transaction not yet complete |
| [`ProductNotAvailableError`](/docs/reference/errors#error-table) | The product is not available in the user's storefront |
| [`StoreProblemError`](/docs/reference/errors#error-table) | The store returned a non-specific error (e.g. billing issue) |
| [`ReceiptInvalidError`](/docs/reference/errors#error-table) | Server-side receipt validation failed |

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import {
      Rovenue,
      PurchaseCancelledError,
      PurchasePendingError,
      ProductNotAvailableError,
      StoreProblemError,
      ReceiptInvalidError,
    } from '@rovenue/react-native-sdk';

    const offerings = await Rovenue.getOfferings();
    const pkg = offerings.current?.packages[0];

    if (pkg) {
      try {
        const result = await Rovenue.purchase(pkg);
        console.log('Purchase succeeded:', result.productId);
        // result.entitlements and result.creditBalance are already up to date
      } catch (e) {
        if (e instanceof PurchaseCancelledError) {
          // User dismissed — no action needed
        } else if (e instanceof PurchasePendingError) {
          // Ask-to-Buy — tell the user to wait for approval
        } else if (e instanceof ReceiptInvalidError) {
          // Show a retry prompt
        } else {
          throw e;
        }
      }
    }
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    let offerings = try await Rovenue.shared.getOfferings()
    if let pkg = offerings.current?.packages.first {
      do {
        let result = try await Rovenue.shared.purchase(pkg)
        print("Purchase succeeded: \(result.productId)")
      } catch is PurchaseCancelledError {
        // User dismissed — no action needed
      } catch is PurchasePendingError {
        // Ask-to-Buy — tell the user to wait for approval
      } catch Rovenue.Error.receiptInvalid {
        // Show a retry prompt
      }
    }
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    val offerings = Rovenue.shared.getOfferings()
    val pkg = offerings.current?.packages?.firstOrNull()

    if (pkg != null) {
      try {
        val result = Rovenue.shared.purchase(pkg)
        Log.d("App", "Purchase succeeded: ${result.productId}")
      } catch (e: PurchaseCancelledException) {
        // User dismissed — no action needed
      } catch (e: PurchasePendingException) {
        // Ask-to-Buy — tell the user to wait for approval
      } catch (e: RovenueException.ReceiptInvalid) {
        // Show a retry prompt
      }
    }
    ```
  </Tab>
</Tabs>

## Account Token

### `getAppAccountToken`

Return a stable UUID tied to the current `rovenueId`. Pass it to StoreKit's `.appAccountToken(uuid)` or Play Billing's `setObfuscatedAccountId()` before initiating a purchase so the store can attribute the transaction to the correct Rovenue subscriber. The token changes when `identify()` is called with a new user.

**Params** — none

**Returns** — `string` (UUID v4)

**Throws** — [`StorageError`](/docs/reference/errors#error-table) if MMKV is unavailable.

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">

    ```ts
    // Pass to your custom StoreKit integration if needed.
    // Most apps don't need to call this directly — purchase() handles it.
    ```
  </Tab>
  <Tab value="Swift">

**Returns** — `void`

**Throws** — nothing

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">

## Logging

### `setLogHandler`

Install a custom log handler to receive structured SDK log entries. Useful for routing SDK logs into your own logging infrastructure. On React Native, returns an unsubscribe function.

**Params**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `handler` | `(entry: LogEntry) => void` | Yes | Callback invoked for each SDK log entry. |

**Returns** — `() => void` on React Native (call to stop receiving logs); `void` on Swift and Kotlin (set to `nil` / `null` to detach).

**Throws** — nothing

The [`LogEntry`](/docs/reference/types#logentry) shape:

```ts
type LogEntry = {
  level: 'debug' | 'info' | 'warn' | 'error';
  message: string;
  timestamp: string; // ISO 8601
};
```

→ Full type reference: [`LogEntry`](/docs/reference/types#logentry)

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    const unsub = Rovenue.setLogHandler((entry) => {
      console.log(`[Rovenue ${entry.level}] ${entry.message}`);
    });

    // Later, to stop receiving logs:
    unsub();
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    Rovenue.shared.setLogHandler { entry in
        print("[Rovenue \(entry.level)] \(entry.message)")

    // To detach:
    Rovenue.shared.setLogHandler(nil)
    ```
  </Tab>
  <Tab value="Kotlin">

  </Tab>

## Change Stream

Subscribe to real-time state change notifications from the SDK. The SDK fires an event whenever the entitlement cache, identity, or credit balance is updated. React to these events to keep your UI in sync without polling.

| Method / Property | Platform | Description |
|-------------------|----------|-------------|
| `addChangeListener(handler)` | React Native | Register a callback; returns an unsubscribe function. |
| `changes` | Swift | `AsyncStream<ChangeEvent>` — iterate with `for await`. |
| `changes` | Kotlin | `SharedFlow<ChangeEvent>` — collect with `.onEach { }.launchIn(scope)`. |

The [`ChangeEvent`](/docs/reference/types#changeevent) values:

| Value | When it fires |
|-------|--------------|
| `EntitlementsChanged` | After any entitlement cache refresh (purchase, `refreshEntitlements()`, foreground sync). |
| `IdentityChanged` | After `identify()` succeeds or `logOut()` resets the user. |
| `CreditBalanceChanged` | After `refreshCredits()` or a successful `consumeCredits()` call. |

→ Full type reference: [`ChangeEvent`](/docs/reference/types#changeevent)

<Tabs groupId="platform" persist items={['React Native', 'Swift', 'Kotlin']}>
  <Tab value="React Native">
    ```ts
    import { Rovenue } from '@rovenue/react-native-sdk';

    const unsub = Rovenue.addChangeListener((event) => {
      switch (event) {
        case 'EntitlementsChanged':
          // Re-read entitlements and update gated UI
          break;
        case 'IdentityChanged':
          // User identity has changed — re-read currentUser()
          break;
        case 'CreditBalanceChanged':
          // Re-read creditBalance() and update the balance display
          break;
      }
    });

    // When your component unmounts:
    unsub();
    ```
  </Tab>
  <Tab value="Swift">
    ```swift
    // Typically in a Task {} inside your SwiftUI view or view model:
    for await event in Rovenue.shared.changes {
        switch event {
        case .entitlementsChanged:
            // Re-read entitlements and update gated UI
        case .identityChanged:
            // User identity has changed — re-read currentUser()
        case .creditBalanceChanged:
            // Re-read creditBalance() and update the balance display
        }
    }
    ```
  </Tab>
  <Tab value="Kotlin">
    ```kotlin
    // In a ViewModel or LifecycleCoroutineScope:
    Rovenue.shared.changes
        .onEach { event ->
            when (event) {
                ChangeEvent.ENTITLEMENTS_CHANGED   -> { /* re-read entitlements */ }
                ChangeEvent.IDENTITY_CHANGED       -> { /* user identity changed */ }
                ChangeEvent.CREDIT_BALANCE_CHANGED -> { /* re-read balance */ }
            }
        }
        .launchIn(viewModelScope)
    ```
  </Tab>
</Tabs>

**See also:** [Reacting to changes guide](/docs/guides/reacting-to-changes)
