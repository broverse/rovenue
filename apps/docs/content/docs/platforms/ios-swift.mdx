<Steps>

<Step>
### Install the SDK

**Swift Package Manager** — in Xcode choose **File → Add Package Dependencies** and enter the repository URL, or add it to your `Package.swift`:

```swift
dependencies: [
  .package(url: "https://github.com/rovenue/sdk-swift.git", from: "0.6.0"),
],
targets: [
  .target(name: "YourApp", dependencies: ["Rovenue"]),
]
```

**CocoaPods** — add to your `Podfile` and run `pod install`:

```ruby
pod 'Rovenue'
```
</Step>

<Step>
### Configure the SDK

Call `Rovenue.configure` once, before any SDK method is used. `AppDelegate.application(_:didFinishLaunchingWithOptions:)` is the correct place:

```swift
import UIKit
import Rovenue

@main
class AppDelegate: UIResponder, UIApplicationDelegate {

  func application(
    _ application: UIApplication,
    didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?
  ) -> Bool {
    Rovenue.configure(
      apiKey: "rov_pub_YOUR_API_KEY"
    )
    return true
  }
}
```

<Callout type="info">
**Self-hosting?** `baseUrl` is optional and defaults to the hosted endpoint `https://api.rovenue.io`. Override it to point at your own deployment, e.g. `Rovenue.configure(apiKey: "rov_pub_YOUR_API_KEY", baseUrl: "https://api.acme.com")`. The host must use `https://` — plain `http://` is accepted only for `localhost` during local development.
</Callout>

The SDK reads `CFBundleShortVersionString` from the main bundle automatically — you do not need to pass `appVersion` manually.

### Types on this page

```swift
// Swift mirror of RovenueConfig
struct RovenueConfig {
  var apiKey: String       // public SDK key — prefix rov_pub_
  var baseUrl: String?     // optional — defaults to https://api.rovenue.io
  var debug: Bool          // default false
  var appVersion: String?  // override auto-detected version
}
```

| Field | Type | Description |
|-------|------|-------------|
| `apiKey` | `String` | Public SDK key from your Rovenue dashboard. Prefix `rov_pub_`. |
| `baseUrl` | `String?` | Optional. Defaults to `https://api.rovenue.io`. Override only when self-hosting (`https://`; `http://` allowed for `localhost` only). |
| `debug` | `Bool` | Enables verbose SDK logs. Do not ship enabled in production. |
| `appVersion` | `String?` | Overrides the version auto-read from the bundle. |

<Callout type="warn">
Call `configure` exactly once before any other SDK call. The ideal location is `applicationDidFinishLaunching`. Calling it after the first SDK method is invoked will throw `RovenueError.notConfigured`.
</Callout>
</Step>

<Step>
### Wire foreground signals

The SDK uses `setForeground` to drive its internal polling cadence. Wire it from your `SceneDelegate`:

```swift
import UIKit
import Rovenue

class SceneDelegate: UIResponder, UIWindowSceneDelegate {

  func sceneWillEnterForeground(_ scene: UIScene) {
    Rovenue.shared.setForeground(true)
  }

  func sceneDidEnterBackground(_ scene: UIScene) {
    Rovenue.shared.setForeground(false)
  }
}
```

<Callout type="info">
If your app uses the SwiftUI `@main` lifecycle rather than a `SceneDelegate`, observe `scenePhase` from the environment and call `setForeground` from `.onChange(of: scenePhase)`.
</Callout>
</Step>

</Steps>

## Purchases

Use `getOfferings()` to fetch products configured in your Rovenue dashboard, then `purchase(_:)` to drive the full StoreKit 2 purchase flow. The SDK validates the receipt server-side and resolves with updated entitlements.

```swift
let offerings = try await Rovenue.shared.getOfferings()
if let pkg = offerings.current?.packages.first {
  do {
    let result = try await Rovenue.shared.purchase(pkg)
    // result.entitlements and result.creditBalance are already up to date
  } catch Rovenue.Error.purchaseCancelled {
    // User dismissed the sheet — no action needed
  } catch Rovenue.Error.purchasePending {
    // Ask-to-Buy / deferred — inform the user and wait for a change event
  }
}

// Re-validate all prior transactions (e.g. after reinstall):
try await Rovenue.shared.restorePurchases()
```

```swift
// Swift mirror of PurchaseResult
struct PurchaseResult {
  var entitlements: [Entitlement]
  var creditBalance: Int
  var productId: String
  var storeTransactionId: String
}

struct Entitlement {
  var id: String
  var isActive: Bool
  var productIdentifier: String
  var store: String
  var expiresIso: String? // ISO 8601 — present for time-limited entitlements
}
```

| Field | Type | Description |
|-------|------|-------------|
| `entitlements` | `[Entitlement]` | Updated entitlement list after the purchase. |
| `creditBalance` | `Int` | Updated credit balance after the purchase. |
| `productId` | `String` | The product identifier that was purchased. |
| `storeTransactionId` | `String` | The StoreKit transaction ID. |

<Callout type="info">
See [Processing Purchases](/docs/guides/processing-purchases) for the full API reference including all error types and the `StoreProduct` shape.
</Callout>

## Observing changes

`Rovenue.shared.changes` is an `AsyncStream` of change events. Consume it inside a long-lived `Task`:

```swift
import Rovenue

Task {
  for await event in Rovenue.shared.changes {
    // event: ChangeEvent (.entitlementsChanged | .identityChanged | .creditBalanceChanged)
    print("Rovenue change:", event)
    // Update your UI or local state here
  }
}
```

Cancel the `Task` (or call `Rovenue.shared.shutdown()`) when the owning object is deallocated.

<Callout type="info">
See [Reacting to Changes](/docs/guides/reacting-to-changes) for a complete guide on handling live subscription events.
</Callout>

## Debugging

Register a log handler to receive structured log entries from the SDK. Set it up before `Rovenue.configure` to capture initialisation logs:

```swift
Rovenue.shared.setLogHandler { entry in
  print("[\(entry.level)] \(entry.message)")
}
```
