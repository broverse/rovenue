<Step>
### Add the dependency

Add the SDK to your module-level `build.gradle.kts`:

```kotlin
dependencies {
  implementation("dev.rovenue:sdk:0.6.0")
}
```

Or in Groovy DSL (`build.gradle`):

```groovy
dependencies {
  implementation 'dev.rovenue:sdk:0.6.0'
}
```

<Callout type="info">
The SDK drives Play Billing natively. Receipt processing and server-side validation happen automatically via the Rovenue API — you do not need to call Play Billing directly.
</Callout>
</Step>

<Step>
### Configure the SDK

Call `Rovenue.configure` once in your `Application.onCreate()`, before any component starts:

```kotlin
import android.app.Application
import dev.rovenue.sdk.Rovenue

class App : Application() {

  override fun onCreate() {
    super.onCreate()

    val versionName = packageManager
      .getPackageInfo(packageName, 0)
      .versionName

    Rovenue.configure(
      apiKey = "rov_pub_YOUR_API_KEY",
      appVersion = versionName,
    )
  }
}
```

<Callout type="info">
**Self-hosting?** `baseUrl` is optional and defaults to the hosted endpoint `https://api.rovenue.io`. Override it to point at your own deployment, e.g. `Rovenue.configure(apiKey = "rov_pub_YOUR_API_KEY", baseUrl = "https://api.acme.com", appVersion = versionName)`. The host must use `https://` — plain `http://` is accepted only for `localhost` during local development.
</Callout>

Register `App` in your `AndroidManifest.xml`:

```xml
<application android:name=".App" ...>
```

### Types on this page

```kotlin
// Kotlin mirror of RovenueConfig
data class RovenueConfig(
  val apiKey: String,          // public SDK key — prefix rov_pub_
  val baseUrl: String? = null, // optional — defaults to https://api.rovenue.io
  val debug: Boolean = false,
  val appVersion: String? = null,
)
```

| Field | Type | Description |
|-------|------|-------------|
| `apiKey` | `String` | Public SDK key from your Rovenue dashboard. Prefix `rov_pub_`. |
| `baseUrl` | `String?` | Optional. Defaults to `https://api.rovenue.io`. Override only when self-hosting (`https://`; `http://` allowed for `localhost` only). |
| `debug` | `Boolean` | Enables verbose SDK logs. Default `false`. |
| `appVersion` | `String?` | Overrides the version read from the package manager. |

<Callout type="warn">
Call `configure` in `Application.onCreate` — not inside an `Activity`. Calling SDK methods before `configure` throws `RovenueException.NotConfigured`.
</Callout>
</Step>

<Step>
### Wire foreground signals

Use `ProcessLifecycleOwner` from `androidx.lifecycle` to observe the process-level foreground state. Add the observer in `Application.onCreate`, after the `configure` call:

```kotlin
import android.app.Application
import androidx.lifecycle.DefaultLifecycleObserver
import androidx.lifecycle.LifecycleOwner
import androidx.lifecycle.ProcessLifecycleOwner
import dev.rovenue.sdk.Rovenue

class App : Application() {

  override fun onCreate() {
    super.onCreate()

    val versionName = packageManager
      .getPackageInfo(packageName, 0)
      .versionName

    Rovenue.configure(
      apiKey = "rov_pub_YOUR_API_KEY",
      appVersion = versionName,
    )

    ProcessLifecycleOwner.get().lifecycle.addObserver(
      object : DefaultLifecycleObserver {
        override fun onStart(owner: LifecycleOwner) {
          Rovenue.shared.setForeground(true)
        }
        override fun onStop(owner: LifecycleOwner) {
          Rovenue.shared.setForeground(false)
        }
      }
    )
  }
}
```

<Callout type="info">
`ProcessLifecycleOwner` fires `onStart`/`onStop` at the process level — not per-Activity. This is the correct granularity for the SDK's polling cadence.
</Callout>
</Step>

</Steps>

## Purchases

Use `getOfferings()` to fetch products configured in your Rovenue dashboard, then `purchase(pkg)` to drive the full Play Billing flow. The SDK validates the receipt server-side and resolves with updated entitlements.

```kotlin
import dev.rovenue.sdk.Rovenue
import dev.rovenue.sdk.PurchaseCancelledException
import dev.rovenue.sdk.PurchasePendingException

val offerings = Rovenue.shared.getOfferings()
val pkg = offerings.current?.packages?.firstOrNull()

if (pkg != null) {
  try {
    // `activity` is your current Activity — Play Billing requires it.
    val result = Rovenue.shared.purchase(activity, pkg)
    // result.entitlements and result.creditBalance are already up to date
  } catch (e: PurchaseCancelledException) {
    // User dismissed the store sheet — no action needed
  } catch (e: PurchasePendingException) {
    // Deferred — inform the user and wait for a change event
  }
}

// Re-validate all prior transactions (e.g. after reinstall):
Rovenue.shared.restorePurchases(activity)
```

```kotlin
// Kotlin mirror of PurchaseResult
data class PurchaseResult(
  val entitlements: List<Entitlement>,
  val creditBalance: Int,
  val productId: String,
  val storeTransactionId: String,

data class Entitlement(
  val id: String,
  val isActive: Boolean,
  val productIdentifier: String,
  val store: String,
  val expiresIso: String?, // ISO 8601 — present for time-limited entitlements
)
```

| Field | Type | Description |
|-------|------|-------------|
| `entitlements` | `List<Entitlement>` | Updated entitlement list after the purchase. |
| `creditBalance` | `Int` | Updated credit balance after the purchase. |
| `productId` | `String` | The product identifier that was purchased. |
| `storeTransactionId` | `String` | The Play Store transaction ID. |

## Observing changes

`Rovenue.shared.changes` is a `SharedFlow` of change events. Collect it from a coroutine scope tied to your component's lifecycle:

```kotlin
import androidx.lifecycle.lifecycleScope
import androidx.lifecycle.repeatOnLifecycle
import androidx.lifecycle.Lifecycle
import dev.rovenue.sdk.Rovenue

// Inside an Activity or Fragment:
lifecycleScope.launch {
  repeatOnLifecycle(Lifecycle.State.STARTED) {
    Rovenue.shared.changes.collect { event ->
      // event: ChangeEvent (ENTITLEMENTS_CHANGED | IDENTITY_CHANGED | CREDIT_BALANCE_CHANGED)
      println("Rovenue change: $event")
    }
  }
}
```

<Callout type="warn">
Collect `Rovenue.shared.changes` from an `Activity` or `ViewModel` scope — not from inside the SDK's own internal scope. Calling `Rovenue.shared.shutdown()` from inside an active collector's coroutine may interrupt the collector mid-flight.
</Callout>

## Suspend functions

SDK methods such as `identify`, `logOut`, `refreshEntitlements`, and `consumeCredits` are suspend functions. Call them from a coroutine:

```kotlin
lifecycleScope.launch {
  // After your auth flow completes:
  Rovenue.shared.identify("user_123")

  // When your user signs out:
  Rovenue.shared.logOut()

  val entitlements = Rovenue.shared.refreshEntitlements()
}
```

<Callout type="info">
See [Identifying Users](/docs/guides/identifying-users) for the full identity model, including what happens when `identify` and `logOut` are called.
</Callout>
