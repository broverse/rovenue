<Steps>

<Step>
### Install the package

Install from npm:

```bash
npm install @rovenue/react-native-sdk
```

**Expo managed workflow** — add the config plugin to `app.json` (or `app.config.js`). It auto-wires the iOS CocoaPods dependency and Android Gradle on prebuild:

```json
{
  "expo": {
    "plugins": ["@rovenue/react-native-sdk"]
  }
}
```

Then run:

```bash
npx expo prebuild
```

**Bare React Native** — after installing, run pod install for iOS:

```bash
cd ios && pod install
```

<Callout type="info">
Peer dependencies: `expo >=51 <53`, `expo-modules-core`, `react >=18`, `react-native >=0.73`.
</Callout>
</Step>

<Step>
### Configure the SDK

Call `Rovenue.configure` once at the root of your app — before any hooks are used. `App.tsx` is the right place:

```tsx
import { Rovenue } from '@rovenue/react-native-sdk';

Rovenue.configure({
  apiKey: 'rov_pub_YOUR_API_KEY',
  baseUrl: 'https://edge.rovenue.app',
});

export default function App() {
  // Foreground wiring added in the next step
  return <RootNavigator />;
}
```

### Types on this page

```ts
type RovenueConfig = {
  apiKey: string;      // public SDK key — prefix rov_pub_
  baseUrl: string;     // always https://edge.rovenue.app
  debug?: boolean;     // verbose SDK logs (dev only)
  appVersion?: string; // override auto-detected version
};
```

| Field | Type | Description |
|-------|------|-------------|
| `apiKey` | `string` | Public SDK key from your Rovenue dashboard. Prefix `rov_pub_`. |
| `baseUrl` | `string` | Always `https://edge.rovenue.app` unless self-hosting. |
| `debug` | `boolean` | Enables verbose SDK logs. Do not ship enabled in production. |
| `appVersion` | `string` | Overrides the version auto-read from the bundle. |

<Callout type="warn">
Call `configure` exactly once, before rendering any component that calls a Rovenue hook. Calling it more than once after initialisation is a no-op but logs a warning.
</Callout>
</Step>

<Step>
### Wire foreground signals

The SDK uses `setForeground` to manage its internal polling cadence. Wire it to React Native's `AppState` inside your root component:

```tsx
import { useEffect } from 'react';
import { AppState } from 'react-native';
import { Rovenue } from '@rovenue/react-native-sdk';

export default function App() {
  useEffect(() => {
    // Signal foreground on mount
    Rovenue.setForeground(true);

    const sub = AppState.addEventListener('change', (state) => {
      Rovenue.setForeground(state === 'active');
    });

    return () => {
      sub.remove();
    };
  }, []);

  return <RootNavigator />;
}
```

<Callout type="info">
`Rovenue.shutdown()` stops all background work (polling, in-flight refreshes). Reserve it for full app teardown — do **not** call it on every background transition. `setForeground(false)` already pauses polling when the app backgrounds.
</Callout>
</Step>

</Steps>

## React hooks

All hooks must be called from components rendered after `configure` has been called.

| Hook | Returns |
|------|---------|
| `useCurrentUser()` | `User \| null` — the identified subscriber, or `null` |
| `useEntitlement(id)` | `Entitlement \| null` — entitlement object for the given ID |
| `useEntitlements()` | `Entitlement[]` — all active entitlements |
| `useCreditBalance()` | `number` — current credit balance |

```ts
type User = {
  rovenueId: string;        // permanent per-install id (prefix rov_), always present
  appUserId: string | null; // your backend user id, null until identify()
};

type Entitlement = {
  id: string;                // e.g. "pro", "credits_500"
  isActive: boolean;
  productIdentifier: string;
  store: string;             // "app_store" | "play_store" | "stripe"
  expiresIso?: string;       // ISO 8601 — present for time-limited entitlements
};
```

| Field | Type | Description |
|-------|------|-------------|
| `rovenueId` | `string` | Permanent per-install id (prefix `rov_`); never changes. |
| `appUserId` | `string \| null` | Your backend user id; `null` until `identify()`. |
| `id` | `string` | Entitlement identifier configured in your dashboard. |
| `isActive` | `boolean` | `true` if the entitlement is currently active. |
| `expiresIso` | `string` | ISO 8601 expiry; present for time-limited entitlements. |

### Example — gating on an entitlement

```tsx
import { useEntitlement } from '@rovenue/react-native-sdk';

export function ProBadge() {
  const pro = useEntitlement('pro');

  if (!pro?.isActive) {
    return null;
  }

  return <Text>Pro member</Text>;
}
```

### Example — displaying credit balance

```tsx
import { useCreditBalance } from '@rovenue/react-native-sdk';

export function CreditsLabel() {
  const balance = useCreditBalance();
  return <Text>{balance} credits</Text>;
}
```

<Callout type="info">
See [Entitlements](/docs/guides/entitlements) and [Credits](/docs/guides/credits) for the full guide on reading and reacting to subscription state.
</Callout>

## Purchases

Use `getOfferings()` to fetch products configured in your Rovenue dashboard, then `purchase(pkg)` to drive the full purchase flow. The SDK triggers the native StoreKit 2 / Play Billing sheet, validates the receipt server-side, and resolves with updated entitlements and credit balance.

```tsx
import {
  Rovenue,
  PurchaseCancelledError,
  PurchasePendingError,
} from '@rovenue/react-native-sdk';

const offerings = await Rovenue.getOfferings();
const pkg = offerings.current?.packages[0];

if (pkg) {
  try {
    const result = await Rovenue.purchase(pkg);
    // result.entitlements and result.creditBalance are already up to date
  } catch (e) {
    if (e instanceof PurchaseCancelledError) {
      // User dismissed the store sheet — no action needed
    } else if (e instanceof PurchasePendingError) {
      // Ask-to-Buy / deferred — inform the user and wait for a change event
    } else {
      throw e;
    }
  }
}
```

To re-validate all prior transactions (e.g. after a reinstall):

```ts
await Rovenue.restorePurchases();
```

```ts
type PurchaseResult = {
  entitlements: Entitlement[];
  creditBalance: number;
  productId: string;
  storeTransactionId: string;
};
```

| Field | Type | Description |
|-------|------|-------------|
| `entitlements` | `Entitlement[]` | Updated entitlement list after the purchase. |
| `creditBalance` | `number` | Updated credit balance after the purchase. |
| `productId` | `string` | The product identifier that was purchased. |
| `storeTransactionId` | `string` | The store-side transaction ID. |

<Callout type="info">
See [Processing Purchases](/docs/guides/processing-purchases) for the full API reference including all error types and the `StoreProduct` shape.
</Callout>

## Change listener

Use `Rovenue.addChangeListener` to react to server-pushed entitlement or credit updates. It returns an unsubscribe function — call it on cleanup.

```tsx
import { useEffect } from 'react';
import { Rovenue } from '@rovenue/react-native-sdk';

useEffect(() => {
  const unsub = Rovenue.addChangeListener((event) => {
    // event: 'EntitlementsChanged' | 'IdentityChanged' | 'CreditBalanceChanged'
    console.log('Rovenue change:', event);
  });

  return () => {
    unsub();
  };
}, []);
```

<Callout type="info">
See [Reacting to Changes](/docs/guides/reacting-to-changes) for a complete guide on handling live subscription events.
</Callout>
