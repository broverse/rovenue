### Files changed

### Change made

**Before:**
```typescript
if (err instanceof ZodError) {
  return c.json(fail(ERROR_CODE.VALIDATION_ERROR, err.message), 400);

```

**After:**
```typescript
if (err instanceof ZodError) {
  log.warn("validation error", {
    issues: err.issues.map((i) => ({ path: i.path.join("."), code: i.code })),
  });
  return c.json(
    fail(ERROR_CODE.VALIDATION_ERROR, "Request validation failed"),
```

### Tests (3 tests, all green)

- Raw response body contains no field names, no `issues`, no `ZodError`

Note: `@hono/zod-validator` uses `safeParseAsync` and returns its own 400 response inline — it does NOT trigger the global errorHandler. The W4.4 handler fires when route code calls `schema.parse()` directly (found in `webhook-verify.ts`, `subscribers.ts`, `experiments.ts`). Tests use a route with direct `schema.parse()` to verify the handler.

## Fix A — validate wrapper: route ALL zod validation failures through errorHandler

### Problem
W4.4 added a ZodError branch to `errorHandler` but `@hono/zod-validator`'s `zValidator()` catches ZodErrors internally and returns its own `{ success: false, error: <ZodError> }` 400 — bypassing `errorHandler`. Only routes with a custom throw-on-failure hook (e.g. `events.ts`) were actually fixed.

### Solution
Created `apps/api/src/lib/validate.ts` — a thin wrapper that passes a hook to `zv()` which throws the ZodError on failure, routing it to `errorHandler`.

```typescript
export const validate = <T extends ZodSchema>(
  target: "json" | "query" | "param" | "form" | "header",
  schema: T,
) =>
  zv(target, schema, (result) => {
    if (!result.success) throw result.error;
  });
```

### Migration
Migrated **45 files** (all `zValidator(` call sites in `apps/api/src`, non-test). Special cases:
- `routes/v1/events.ts`: existing throw-hook collapsed into `validate()` (consolidated)
- `routes/dashboard/subscribers.ts`: `throwOnInvalid` helper (leaked field names via `errors[0]?.message`) removed; replaced with `validate()`

No remaining `zValidator(` calls in production code. The only `@hono/zod-validator` import is in `lib/validate.ts` itself.

### Tests updated (7 test files)
All needed `app.onError(errorHandler)` added to `buildApp()` since ZodErrors are now thrown (not auto-handled inline):
1. `routes/public/funnels.test.ts` — W4.3e (answer string > 2000 chars → 400)
2. `routes/v1/sdk-sessions.test.ts` — malformed body → 400
3. `routes/dashboard/credits.integration.test.ts` — non-positive amount → 400
4. `routes/dashboard/invitations.integration.test.ts` — role=OWNER rejection → 400
5. `routes/dashboard/members.integration.test.ts` — role=OWNER rejection → 400
6. `routes/dashboard/subscriptions.integration.test.ts` — Zod query validation → 400
7. `routes/dashboard/products.store-catalog.test.ts` — store=web rejection → 400

No body-shape assertions changed (all tests asserted only on `status 400`, not the raw zValidator body).

## Fix B — bound record key count in answerValueSchema

Added `.refine((obj) => Object.keys(obj).length <= 100, { message: "too many keys" })` to the `z.record(...)` branch so records are bounded symmetrically with arrays (`.max(100)`).

## Fix C — funnel rate limiters: use default clientIp

Removed `identify: firstHopIp` from both `endpointRateLimit` calls in `funnels.ts`. The `firstHopIp` helper read the leftmost XFF hop (spoofable). The default `identify` in `endpointRateLimit` uses `clientIp()` which honours `TRUSTED_PROXY_COUNT` and is spoof-resistant.

Removed `firstHopIp` helper function and `import type { Context } from "hono"` (no longer needed).
