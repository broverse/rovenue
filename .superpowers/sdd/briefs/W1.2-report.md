Replaced the unconditional RAISE with a conditional:
- UPDATE → always rejected (ERRCODE `restrict_violation`).

- Added exported helper `withLedgerDeleteAuthorized<T>(db, fn)` — wraps `fn` in a transaction that sets `SET LOCAL "rovenue.allow_ledger_delete" = 'on'` before executing, authorizing DELETEs for that transaction only. UPDATE remains forbidden inside and outside this helper.

# W1.2 deleteProject Cascade Fix

## Status: DONE

## Commit
`4e6c5537` — `fix(db): authorize credit_ledger cascade DELETE in deleteProject`

## Problem
`deleteProject` did a bare `db.delete(projects)` which cascades to `credit_ledger`. The append-only trigger rejects any DELETE unless the transaction has `SET LOCAL "rovenue.allow_ledger_delete" = 'on'`, so any project with ledger rows would fail at runtime.

## Fix
Added `await db.execute(sql\`SET LOCAL "rovenue.allow_ledger_delete" = 'on'\`)` at the top of `deleteProject` before the delete. This reuses the caller's existing transaction (the dashboard route wraps it in `drizzle.db.transaction`); `SET LOCAL` is scoped to the current transaction so the flag does not leak.

## Files Changed
- `packages/db/src/drizzle/repositories/projects.ts` — added SET LOCAL before `db.delete(projects)` in `deleteProject`
- `packages/db/src/drizzle/repositories/projects.delete.integration.test.ts` — new integration test (seeds project + subscriber + VC + ledger row, calls deleteProject, asserts success + rows gone)
